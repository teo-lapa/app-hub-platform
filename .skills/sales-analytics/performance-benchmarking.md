## Regole Critiche

### üéØ REGOLA #1: Comparabilit√† (MASSIMA PRIORIT√Ä!)

**PROBLEMA COMUNE**: Confrontare elementi non comparabili (mele con arance).

**SOLUZIONE**:
1. SEMPRE confrontare periodi di **uguale durata**
2. Normalizzare per **fattori distorsivi**
3. Segmentare correttamente

**Esempi di Confronti NON Validi**:
```
‚ùå Gennaio (31gg) vs Febbraio (28gg) senza normalizzare
‚ùå Venditore full-time vs part-time senza ponderare ore
‚ùå Prodotto lancio 2024 vs prodotto storico senza contesto
‚ùå Store A (citt√†) vs Store B (paese) senza considerare bacino
```

**Normalizzazione Richiesta**:
```javascript
// Per periodi diversi
revenue_per_day = total_revenue / days_in_period

// Per team diverse dimensioni
revenue_per_salesperson = team_revenue / team_size

// Per prodotti diverse fasce
margin_percentage = (revenue - cost) / revenue √ó 100
```

### üìä REGOLA #4: Deviazioni e Varianze

**Analizza SEMPRE la distribuzione**, non solo la media.

**Metriche Distribuzione**:
```javascript
{
  "mean": 100000,           // Media
  "median": 95000,          // Mediana (pi√π robusta a outlier)
  "std_dev": 15000,         // Deviazione standard
  "coefficient_variation": 0.15,  // CV = std_dev / mean
  "min": 60000,
  "max": 180000,
  "q1": 85000,              // 1¬∞ quartile (25%)
  "q3": 115000,             // 3¬∞ quartile (75%)
  "iqr": 30000              // Interquartile range
}
```

**Interpretazione CV** (Coefficient of Variation):
- CV < 0.15: Distribuzione omogenea (team equilibrato)
- CV 0.15-0.30: Variabilit√† moderata (normale)
- CV > 0.30: Alta variabilit√† (gap significativi)

**Esempio**:
```
Team A: Mean ‚Ç¨100k, StdDev ‚Ç¨10k ‚Üí CV=0.10 (omogeneo)
Team B: Mean ‚Ç¨100k, StdDev ‚Ç¨40k ‚Üí CV=0.40 (disomogeneo)
‚Üí Team B ha problemi di performance gap interni!
```

### üìà REGOLA #6: Trend Analysis nel Benchmark

**Non guardare solo snapshot**, analizza il TREND.

**Esempio**:
```
Venditore A: Revenue ‚Ç¨80k (Gen) ‚Üí ‚Ç¨85k (Feb) ‚Üí ‚Ç¨90k (Mar)  [TREND +12.5%]
Venditore B: Revenue ‚Ç¨100k (Gen) ‚Üí ‚Ç¨95k (Feb) ‚Üí ‚Ç¨90k (Mar) [TREND -10%]

Snapshot Marzo: B > A
Trend 3-mesi: A sta migliorando, B sta peggiorando!
```

**Momentum Score**:
```javascript
momentum = (current_period - avg_last_3_periods) / avg_last_3_periods

if (momentum > 0.10) return "strong_upward"
if (momentum > 0.05) return "upward"
if (momentum > -0.05) return "stable"
if (momentum > -0.10) return "downward"
return "strong_downward"
```

## Errori Comuni da Evitare

### Errore #1: Periodi Non Comparabili
```
‚ùå SBAGLIATO: Confrontare Gen (31gg) vs Feb (28gg) direttamente
‚úÖ CORRETTO:  Normalizzare: revenue_per_day o pro-rata
```

### Errore #2: Ignorare Contesto
```
‚ùå SBAGLIATO: Penalizzare new hire per basse vendite prime settimane
‚úÖ CORRETTO:  Segmentare: tenure <3 mesi escluso da ranking generale
```

### Errore #3: Solo Media Senza Distribuzione
```
‚ùå SBAGLIATO: "Team media ‚Ç¨100k" (nasconde gap interni)
‚úÖ CORRETTO:  "Team media ‚Ç¨100k, CV 0.35 - alta variabilit√† interna"
```

### Errore #4: Ranking Senza Momentum
```
‚ùå SBAGLIATO: Solo snapshot posizione corrente
‚úÖ CORRETTO:  Include trend: "Rank 8, ma momentum +0.15 (improving fast)"
```

### Errore #5: Benchmark Competitor Non Verificati
```
‚ùå SBAGLIATO: Usare dati competitor senza validare fonte
‚úÖ CORRETTO:  Indicare data_quality e source per ogni metrica
```

## Note Tecniche

- **Modello**: Claude 3.5 Sonnet (ottimizzato per analisi comparative)
- **Max tokens**: 8192
- **Temperature**: 0 (deterministic - CRITICO per ranking accuracy)
- **Timeout**: 45 secondi

**Algoritmi Statistici**:
- Quartili: Tukey's hinges method
- Outlier detection: IQR method (1.5√óIQR rule)
- Trend detection: Linear regression on 3-month window
- Normalization: Z-score, Min-Max scaling

**Performance**:
- Batch processing per >100 entities
- Caching per benchmark ricorrenti
- Parallel stats calculation
