## Regole Critiche

### üéØ REGOLA #1: Accuratezza Matematica (MASSIMA PRIORIT√Ä!)

**PROBLEMA COMUNE**: Arrotondamenti errati o formule sbagliate portano a KPI fuorvianti.

**SOLUZIONE**:
1. Usa SEMPRE precisione decimale (min 2 cifre)
2. Calcola PRIMA i valori individuali, POI gli aggregati
3. Verifica formule prima di applicarle

**Formule Standard**:
```
Revenue Growth Rate = ((Revenue_Current - Revenue_Previous) / Revenue_Previous) √ó 100
Average Order Value (AOV) = Total_Revenue / Number_of_Orders
Customer Lifetime Value (CLV) = Avg_Purchase_Value √ó Avg_Purchase_Frequency √ó Avg_Customer_Lifespan
Conversion Rate = (Number_of_Conversions / Total_Visitors) √ó 100
Churn Rate = (Customers_Lost / Total_Customers_Start) √ó 100
```

**Esempi**:
```

## Errori Comuni da Evitare

### Errore #1: Percentuali Calcolate Male
```
‚ùå SBAGLIATO: Change% = (Current - Previous) / Current
‚úÖ CORRETTO:  Change% = ((Current - Previous) / Previous) √ó 100
```

### Errore #2: Arrotondamenti Prematuri
```
‚ùå SBAGLIATO: Arrotondare ogni step intermedio
‚úÖ CORRETTO:  Arrotondare SOLO il risultato finale
```

### Errore #3: Ignorare Division by Zero
```
‚ùå SBAGLIATO: AOV = Revenue / 0 = Infinity
‚úÖ CORRETTO:  AOV = null, status: "no_orders"
```

### Errore #4: Periodi Non Comparabili
```
‚ùå SBAGLIATO: Confrontare Gennaio (31gg) con Febbraio (28gg) senza normalizzare
‚úÖ CORRETTO:  Normalizzare per numero giorni o usare rate giornaliero
```

### Errore #5: Valute Miste Senza Conversione
```
‚ùå SBAGLIATO: Sommare ‚Ç¨100 + $100 = 200
‚úÖ CORRETTO:  Convertire prima: ‚Ç¨100 + $100(‚Üí‚Ç¨92) = ‚Ç¨192
