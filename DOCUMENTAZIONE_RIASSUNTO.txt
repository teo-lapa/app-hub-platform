STACK TECNOLOGICO SUMMARY:
================================================================================

Frontend:
  - Next.js 14 (App Router, Server Components)
  - React 18
  - TypeScript 5
  - Tailwind CSS 3
  - Framer Motion (animations)
  - Zustand (state management)
  - React Hot Toast (notifications)

Backend:
  - Node.js 18+
  - Next.js API Routes
  - JWT authentication
  - Odoo RPC client (custom)
  - Vercel Postgres (optional)

DevOps:
  - Git version control
  - npm/pnpm package manager
  - Next.js build system
  - Vercel deployment (recommended)
  - ESLint + TypeScript linting

================================================================================
ODOO INTEGRATION SUMMARY:
================================================================================

Autenticazione:
  - OdooClient.authenticate(login, password)
  - Salva session_id in cookie
  - Auto-refresh via SessionManager

RPC Calls:
  - OdooRPCClient.callKw(model, method, args, kwargs)
  - OdooRPCClient.searchRead(model, domain, fields, limit, order)
  - Supporta create, write, read, delete

Modelli Principali:
  - res.partner (clienti/fornitori)
  - sale.order (ordini)
  - product.product (prodotti)
  - account.move (fatture)
  - crm.team (team vendite)
  - stock.picking (prelievi)

Domain Filters:
  - Supporta operatori: =, !=, >, <, >=, <=, like, ilike, in, etc.
  - Supporta logica: & (AND), | (OR), ! (NOT)

BEST PRACTICES:
================================================================================

1. SEMPRE usare TypeScript
2. SEMPRE validare input server-side
3. SEMPRE gestire errori con try-catch
4. SEMPRE usare environment variables per config
5. SEMPRE proteggere routes richiedendo autenticazione
6. SEMPRE fare API calls via API routes (mai da client)
7. SEMPRE usare SessionManager per auto-reconnect Odoo
8. SEMPRE usare Zustand per state management
9. SEMPRE memoizzare componenti expensive
10. SEMPRE testare con account diversi (free, pro, admin)
